# Nous — географический IRC

Чат, привязанный к месту: весь мир разбит на **вложенные административные каналы**,
и ты автоматически состоишь сразу в нескольких — от квартала до страны. Чем выше
уровень, тем шире аудитория. Модель общения — как зональный чат в MMORPG (WoW, Eve):
канал = место, а не диалог с человеком.

```
🌍 Страна
 └─ 🗺 Область
     └─ 🏙 Город
         └─ 🏘 Район
             └─ 🏠 Квартал
```

Канал переключается командами, как в IRC:

| Команда  | Уровень |
|----------|---------|
| `/world` | страна  |
| `/city`  | город   |
| `/local` | район   |
| `/say`   | квартал |

Гарантированный локальный уровень — **Район**; **Квартал** опционален и появляется
там, где в OpenStreetMap есть данные. Членство в каналах не выбирается вручную — оно
**однозначно выводится из координат**: одни и те же координаты у любых двух клиентов
всегда дают один и тот же набор каналов.

## Этот репозиторий

`nous-meta` — не код, а **общий контекст для всех под-проектов**: продуктовая идея,
модель каналов и **контракт ID каналов** (граница между геокодингом и сервером).
Каждый под-проект подключает его в своём `CLAUDE.md` строкой `@../nous-meta/CLAUDE.md`.

Полное описание модели и контракта — в [`CLAUDE.md`](./CLAUDE.md).

## Под-проекты

| Репозиторий | Роль | Язык |
|---|---|---|
| `nous-research` | Геокодинг: координаты → иерархия каналов (ID) | JS (прототип) |
| `nous-server` | Бэкенд: хранение сообщений и подписки по ID каналов | Go/Scala (план) |
| `nous-client` | Клиент (позже) | — |

## Откуда берутся каналы

Источник данных — **OpenStreetMap / Nominatim** (не Google): открытые ID
(`osm_type/osm_id`), возможность поднять свой Nominatim, ISO 3166-2 для областей
из коробки. ID канала — это стабильный код единицы (`RU`, `RU-MOW`,
`relation/2555133`), а имена («Frankfurt») — только для отображения.
