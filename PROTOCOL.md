# Wire-протокол клиент ↔ сервер (прототип)

**Граница между `nous-client` и `nous-server`.** Как и контракт ID каналов
(см. [`CLAUDE.md`](./CLAUDE.md)), это согласованный формат: обе стороны обязаны
понимать его одинаково, менять — только вместе.

Транспорт — **WebSocket**, эндпоинт `ws://<host>:8080/ws`. Каждое сообщение —
JSON-объект-конверт:

```json
{ "type": "<тип>", "data": { ... } }
```

Прототипная договорённость (не финал): нет аутентификации, нет персистентности,
отправитель идентифицируется свободным ником.

## Клиент → сервер

| `type` | `data` | Смысл |
|---|---|---|
| `hello` | `{ "nick": "alex" }` | задать ник (иначе `anon`) |
| `locate` | `{ "lat": 55.76, "lng": 37.62 }` | сообщить координаты; сервер геокодит, **подписывает** соединение на весь набор каналов и отвечает `located` |
| `publish` | `{ "channel": "RU-MOW", "text": "привет" }` | отправить сообщение в один канал по его ID |

## Сервер → клиент

| `type` | `data` | Смысл |
|---|---|---|
| `located` | `{ "channels": [Channel, ...] }` | набор каналов точки, broad→specific; пустые уровни опущены |
| `message` | `{ "channel": "RU-MOW", "sender": "alex", "text": "привет", "ts": 1719580800000 }` | сообщение в канале, на который клиент подписан (включая собственные) |
| `error` | `{ "code": "bad_data", "message": "..." }` | ошибка обработки кадра |

### Объект `Channel`

```json
{ "id": "relation/2555133", "level": "city", "label": "Город", "name": "Москва" }
```

- `id` — **ID канала** (контракт из `CLAUDE.md`): ISO 3166-1/-2 для страны/области,
  `osm_type/osm_id` для города/района/квартала. Ключ хранения и роутинга.
- `level` — `country | region | city | district | quarter`.
- `label`, `name` — только для отображения.

`ts` — Unix-время в миллисекундах (ставит сервер).

## Поток

1. `→ hello` (опц.) → `→ locate` → `← located` (сервер уже подписал на все каналы набора).
2. Клиент показывает каналы; команды `/world` `/city` `/local` `/say` лишь выбирают
   **активный** канал для просмотра/отправки — подписка остаётся на весь набор.
3. `→ publish {channel, text}` → всем подписчикам канала приходит `← message`.

## Зарезервировано (пока не реализуем)

- переподписка диффом при движении пользователя (только изменившиеся уровни);
- история/догрузка сообщений канала; аутентификация.
